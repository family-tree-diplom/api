
- App
- - Controller (3. Контроллеры используют Repository, Service и Model)
- - Event (8. События, на которые подписаны Channel)
- - Service (4. Основная бизнес логика, используются другие Repository, Service и Entity)

- Domain
- - Channel (8. Каналы позволяют подписывать клиентов на события Event)
- - Entity (7. Сущность содержит аннотации для описания модели данных, по которой ORM генерит схему таблиц БД)
- - Factory (6. Позволяют удобно создавать сущности)
- - Repository (5. Позволяют работать с сущностями в хранилице данных (get, save, create, read, update, delete) использует Event, Entity и Factory)

- Infrastructure
- - API (1. веб-сервер, обрабатывает входящие соединения, отправляет ответ на запрос, выбирает контроллер исходя из роутинга и задаем ему Mapper)
- - Mapper (2. выполяет методы контроллера (метода контроллера(ов) и формирует формат ответа)
- - Model (9. Модели - для работы с СУБД - это как репозитории, только репозитории для работы с одной сущностью, а модель это общий случай, когда разные кастомные запросы, а не обычные)
- - Task (10. Задачи - это классы задач, что выполняются в отдельном процессе асинхронно, используется для того чтобы неминуемо синхронные процессы вынести в отдельный процесс от основного)
